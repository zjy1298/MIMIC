Type 1: Text-Encoded Visual/Spatial Reasoning
1.1 Grid & ASCII Diagram Parsing
● Definition: Layouts represented using character matrices, ASCII art, or coordinate lists to depict two-dimensional structures; tasks depend on spatial properties such as adjacency, connectivity, and shape.
● Distinguishing Feature: Requires parsing the input into a 2D array for reliable processing; understanding local characters is insufficient — global alignment is necessary.
● Examples
  ○ ASCII mazes/maps: # for walls, . for paths, S for start, E for end (even when subsequently using BFS/Dijkstra, the main difficulty often lies in "reading and aligning the diagram")
  ○ "Find the area/perimeter of enclosed regions", "count the number of connected components", "boundary tracing"
  ○ "Identify a shape from ASCII art and output its boundary points / result after rotation"
(Note: This differs from Type 5.1 Path Traversal in that 5.1 assumes the graph structure is already given; the difficulty in 1.1 is first recovering the graph structure from text.)
1.2 Geometric & Relational Grounding
● Definition: Input consists of natural language or symbolic descriptions (left/right/up/down, intersection/containment, angle/distance), requiring strict derivation of spatial relationships or coordinate results.
● Distinguishing Feature: Prone to errors where "language comprehension is fine, but spatial grounding is inconsistent"; code can ensure consistency through geometric constraints, vector operations, and intersection calculations.
● Examples
  ○ "Starting from (0,0), follow a sequence of N/E/S/W moves — find the endpoint / whether the path self-intersects"
  ○ Multi-constraint geometry: line segment intersection, point-in-polygon, minimum bounding rectangle (input given as coordinate text)
  ○ Directional constraint puzzles: A is upper-left of B, C is to the right of A... determine whether there is a contradiction / provide a valid layout
1.3 Glyph / Radical / Stroke Decomposition
● Definition: The problem appears to be about text, but depends on visual properties of Chinese characters such as components, strokes, mirror/rotation, and structural composition (left-right, top-bottom).
● Distinguishing Feature: Requires an enumerable glyph representation (component tree, stroke sequence, font outline); LLMs often guess based on common knowledge, resulting in incomplete coverage and difficulty in verification.
● Examples
  ○ "Decompose the character 'ying (win)' into its components", "add one stroke to form a different character", "swap left and right parts — what character results?"
  ○ Radical substitution rules (e.g., water-radical → speech-radical) to form new characters and determine whether they are common characters
  ○ Glyph puzzles / riddles that depend on visual form rather than semantic meaning
  ○ How many circular shapes are in a character
(To do this well with code typically requires external glyph resources: Unihan, IDS (Ideographic Description Sequences), stroke databases, or font outlines; but once resources are available, execution is deterministic.)
1.4 Layout-sensitive Tables & Diagrams
● Definition: Information resides not only in the characters themselves but also in formatting elements such as alignment, columns, indentation, and spacing.
● Distinguishing Feature: Extremely sensitive to spaces and alignment; LLMs easily "misread" the layout or reformat it, causing semantic changes.
● Examples
  ○ Fixed-width tables (columns aligned with spaces): extracting a specific column, computing summaries
  ○ Tree structures (indentation indicates parent-child relationships): restoring to JSON
  ○ ASCII flowcharts / organizational charts: parsing edge connections
Type 2: Sub-Token / Character Manipulation
Sub-classification Logic: What kind of micro-level processing does the task perform on characters?
● 2.1 Character Counting, Extraction & Replacement
  ○ Definition: Does not alter the text content, only searches for specific letters or symbols within words, or performs replacements.
  ○ Distinguishing Feature: The answer is typically a number, or a subset of the original text.
  ○ Examples: "How many r's are in strawberry?", "Extract all words whose 3rd letter is x from a passage."
● 2.2 Character Mapping & Cryptography
  ○ Definition: Replaces or shifts each letter according to specific rules.
  ○ Distinguishing Feature: The output is a completely transformed new text.
  ○ Examples: "Caesar cipher decryption", "Convert all letters by adding 2 to their ASCII code value", "Base64 encoding/decoding."
● 2.3 Strict Formatting & Regex
  ○ Definition: Validates or reconstructs strings that must conform to extremely rigid symbol combination rules.
  ○ Distinguishing Feature: The focus is on the "legality" of symbol arrangement.
  ○ Examples: "Verify whether these 100 email addresses have correct format", "Write code to find all serial numbers matching the pattern XXX-XXXX-XXX in the text."

Type 3: Precise Mathematical Computation
Sub-classification Logic: Which branch of mathematics does the task belong to?
● 3.1 High-Precision Arithmetic
  ○ Definition: Basic addition, subtraction, multiplication, division, exponentiation, and logarithm operations, but involving numbers beyond what a large model can "memorize."
  ○ Distinguishing Feature: Simply computing "large numbers" or "numbers with extremely many decimal places."
  ○ Examples: "Calculate 9834710924 * 12309128", "Calculate the first 1000 digits of pi."
● 3.2 Algebraic & Symbolic Math
  ○ Definition: Solving equations, calculus, matrix derivation, and other computations requiring algebraic rules.
  ○ Distinguishing Feature: Contains unknowns (x, y), requiring order reduction, matrix inversion, or integration.
  ○ Examples: "Find the roots of a quadratic equation", "Calculate the eigenvalues of a 4x4 matrix", "Find the derivative of the function y=x^3+sin(x)."
● 3.3 Statistics & Set Operations
  ○ Definition: Performing macro-level statistical descriptions or probability calculations on a set of numbers.
  ○ Distinguishing Feature: Involves mean, variance, distribution, permutations and combinations.
  ○ Examples: "Find the standard deviation of these 50 floating-point numbers", "Calculate the exact probability of drawing a straight flush from a deck of cards."

Type 4: Large-Scale Data Processing
Sub-classification Logic: What structured operation does the code need to perform on massive data?
● 4.1 Search & Filtering
  ○ Definition: Finding the very few targets that meet criteria within massive data (needle in a haystack).
  ○ Distinguishing Feature: Input data is extremely large, output data is extremely small.
  ○ Examples: "From 1000 lines of server logs, find all records containing Error 502 that occurred during early morning hours."
● 4.2 Aggregation & Sorting
  ○ Definition: Rearranging or statistically summarizing global data.
  ○ Distinguishing Feature: Must examine the entire dataset; missing even one row will cause ranking or total errors.
  ○ Examples: "Sort this CSV containing 500 customers by spending amount in descending order", "Group by department and calculate average salary."
● 4.3 Structural Transformation & Mapping
  ○ Definition: Changing the hierarchical or organizational form of data without changing the data itself.
  ○ Distinguishing Feature: For example, converting from a tree structure to a flat table, or merging multiple tables.
  ○ Examples: "Merge two complex JSON objects containing hundreds of nodes", "Flatten this nested dictionary and convert to CSV format."

Type 5: Strict Rule & State Simulation
Sub-classification Logic: What algorithmic thinking does the code need to solve this logical problem?
● 5.1 Pathfinding & Graph Traversal
  ○ Definition: Optimization or connectivity determination of spatial relationships.
  ○ Distinguishing Feature: The core algorithm is necessarily DFS (depth-first search), BFS (breadth-first search), A*, or Dijkstra.
  ○ Examples: "Minimum steps through a maze", "Traveling salesman problem (visit all cities with minimum distance)", "Topological sorting."
● 5.2 Constraint Satisfaction & Backtracking
  ○ Definition: Filling a system under given, mutually exclusive rigid rules (if a step conflicts, it must be undone and retried).
  ○ Distinguishing Feature: Seeks a "valid solution"; the core algorithm is Backtracking.
  ○ Examples: "Solve a hard Sudoku puzzle", "Eight Queens problem", "Employee shift scheduling with extremely strict rules."
● 5.3 Iterative State Simulation
  ○ Definition: Given an initial state and transition rules, compute the final state as time "steps" progress.
  ○ Distinguishing Feature: Has a strong temporal order, where the state at step N depends entirely on the state at step N-1 (state machine).
  ○ Examples: "Conway's Game of Life simulated for 100 generations", "Simulate a ball bouncing 10 times and find the final coordinates", "Simulate the register state after executing a piece of assembly code."
