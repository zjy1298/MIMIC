# Task
I will provide you with a [Problem Description]. Your goal is to remove all character names, background stories, and filler text, then describe the core requirements of the problem using the most concise and direct language possible.

# Constraints
1. **Core Only**: Do not include any preamble. Use the direct structure: "Given..., find/calculate...".
2. **Plain Talk**: Use standard programming terminology that developers/contestants understand instantly (e.g., array, subarray, non-overlapping, maximum sum, path, tree, etc.).

4. **Preserve Semantics**: Do not lose or change any logic, constraints, input/output requirements, or the intended solving method of the original problem.
